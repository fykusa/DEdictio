Jak postavit aplikaci?

Existující design.html je plně funkční — otázka je, jak ho rozšířit o reálná CSV data a případně další funkce.

A

Rozšířit design.html — pure vanilla JS

Zachovat jediný HTML soubor. Přidat CSV parser, načíst slovicka.csv přes fetch() a nahradit hardcoded data. Žádný build systém.

Výhody

  • Nulová komplexita — jeden soubor
  • Funguje bez instalace čehokoli
  • Jednoduché nasazení

Nevýhody

  • Při růstu funkcí se soubor stane nepřehledný
  • CSV musí být přístupné přes server (ne přímo z disku)
B

Vite + vanilla JS — modulární, bez frameworku ⭐ doporučeno

Rozdělit do modulů: data.js (CSV parser), quiz.js (logika), main.js (UI). Vite builduje do statických souborů. Styl zůstane identický s designem.

Výhody

  • Čistá modulární struktura
  • Hot reload při vývoji
  • Snadné nasazení kamkoli (GitHub Pages, Netlify...)
  • Žádný framework — design se nemusí přepisovat

Nevýhody

  • Potřeba Node.js + npm
C

React / Vue — komponentový framework

Přepsat do moderního frameworku. Každá obrazovka = komponenta. Silný state management.

Výhody

  • Nejlepší škálovatelnost do budoucna

Nevýhody

  • Overkill pro tuto velikost projektu
  • Existující CSS/HTML design se musí přepsat
  • Větší bundle, složitější setup
